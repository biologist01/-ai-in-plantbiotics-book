# مینی پروجیکٹ: بیماری کی پیش گوئی کا نظام
## تعارف
دنیا کو خوراک کی سلامتی کو یقینی بنانے کے لیے ایک بڑا چیلنج کا سامنا ہے۔ فصلیں کی بیماریاں خوراک کی پیداوار کے لیے ایک بڑا خطرہ ہیں، جو پیداوار اور معیار میں نمایاں نقصان کا سبب بنتی ہیں۔ بیماری کی ابتدائی تشخیص مؤثر انتظام اور بیماری کے پھیلاؤ کی روک تھام کے لیے ضروری ہے۔ مصنوعی ذہانت (AI) اور مشین لرننگ (ML) کے عروج کے ساتھ، اب ہم پیش گوئی کے ماڈل تیار کر سکتے ہیں جو ابتدائی مرحلے پر بیماریوں کی تشخیص کر سکتے ہیں، جو کسانوں کو بیماری کے پھیلاؤ کو روکنے کے لیے پیشگی اقدامات کرنے کی اجازت دیتا ہے۔ اس ماڈیول میں، ہم سنسور ڈیٹا اور ماحولیاتی عوامل کا استعمال کرتے ہوئے ابتدائی بیماری کی تشخیص کے لیے مکمل مشین لرننگ سسٹم تیار کریں گے۔

## بنیادی تصورات
### پروجیکٹ کا جائزہ: ابتدائی بیماری کی انتباہی نظام
اس پروجیکٹ کا مقصد ایک بیماری کی پیش گوئی کا نظام تیار کرنا ہے جو فصلیں میں ابتدائی مرحلے پر بیماریوں کی تشخیص کر سکتا ہے۔ یہ نظام سنسور ڈیٹا اور ماحولیاتی عوامل جیسے کہ درجہ حرارت، نمی، اور مٹی کی نمی کا استعمال کرتے ہوئے بیماری کے وقوع کی امکانات کی پیش گوئی کرے گا۔ یہ نظام درج ذیل اجزاء پر مشتمل ہوگا:

* ڈیٹا کی جمع: سنسورز اور ماحولیاتی عوامل سے ڈیٹا کی جمع
* ڈیٹا کی پری پروسیسنگ: جمع کیے گئے ڈیٹا کی صفائی اور پری پروسیسنگ
* فیچر انجینئرنگ: پری پروسس کیے گئے ڈیٹا سے متعلقہ خصوصیات کی اختراع
* ماڈل کی تربیت: انجینئرڈ خصوصیات کا استعمال کرتے ہوئے مشین لرننگ ماڈل کی تربیت
* ماڈل کی تعیناتی: تربیت یافتہ ماڈل کی تعیناتی پروڈکشن ماحول میں
* ریئل ٹائم مانیٹرنگ: نظام کی ریئل ٹائم مانیٹرنگ بیماریوں کی تشخیص کے لیے

### ڈیٹا سیٹ کی جمع اور تیاری
بیماری کی پیش گوئی کے نظام کو بنانے کے لیے، ہم ایک ڈیٹا سیٹ کی ضرورت ہے جو فصلیں، سنسور ڈیٹا، اور ماحولیاتی عوامل کے بارے میں معلومات پر مشتمل ہو۔ ڈیٹا سیٹ میں درج ذیل خصوصیات شامل ہونی چاہیئں:

| خصوصیت | وضاحت |
| --- | --- |
| فصلیں کی قسم | فصلیں کی قسم (جیسے کہ گندم، چاول، ٹماٹر) |
| درجہ حرارت | سنسور سے درجہ حرارت کی پڑھائی |
| نمی | سنسور سے نمی کی پڑھائی |
| مٹی کی نمی | سنسور سے مٹی کی نمی کی پڑھائی |
| بیماری کا مقام | فصلیں میں بیماری کا مقام ہے یا نہیں |

ہم اس ڈیٹا کو مختلف ذرائع سے جمع کر سکتے ہیں، جیسے کہ:

* سنسور ڈیٹا فارمز سے
* ماحولیاتی ڈیٹا موسمی اسٹیشنوں سے
* فصلیں کی ڈیٹا زرعی ڈیٹابیس سے

یہاں ایک مثال ہے کہ ہم کس طرح Python کا استعمال کرتے ہوئے ڈیٹا سیٹ کو جمع اور تیار کر سکتے ہیں:
```python
import pandas as pd

# ڈیٹا سیٹ کو لوڈ کریں
df = pd.read_csv('fasilay_data.csv')

# ڈیٹا کی صفائی اور پری پروسیسنگ
df = df.dropna()  # خالی مقامات کو ہٹا دیں
df = df.drop_duplicates()  # دہرے مقامات کو ہٹا دیں

# ڈیٹا کو تربیت اور ٹیسٹ سیٹ میں تقسیم کریں
from sklearn.model_selection import train_test_split
X_train, X_test, y_train, y_test = train_test_split(df.drop('bimari_ki_jaga', axis=1), df['bimari_ki_jaga'], test_size=0.2, random_state=42)
```

### سنسور ڈیٹا سے خصوصیات کی اختراع
خصوصیات کی اختراع سنسور ڈیٹا سے متعلقہ خصوصیات کی اختراع کا عمل ہے۔ اس میں، ہم خصوصیات جیسے کہ:

* گزشتہ ہفتے کے دوران اوسط درجہ حرارت
* گزشتہ ہفتے کے دوران زیادہ سے زیادہ نمی
* گزشتہ ہفتے کے دوران کم سے کم مٹی کی نمی

یہاں ایک مثال ہے کہ ہم کس طرح Python کا استعمال کرتے ہوئے ان خصوصیات کو اختراع کر سکتے ہیں:
```python
import numpy as np

# سنسور ڈیٹا سے خصوصیات کی اختراع
X_train['avg_temp'] = X_train['drj_hararat'].rolling(window=7).mean()
X_train['max_nami'] = X_train['nami'].rolling(window=7).max()
X_train['min_mati_nami'] = X_train['mati_nami'].rolling(window=7).min()

X_test['avg_temp'] = X_test['drj_hararat'].rolling(window=7).mean()
X_test['max_nami'] = X_test['nami'].rolling(window=7).max()
X_test['min_mati_nami'] = X_test['mati_nami'].rolling(window=7).min()
```

### کثیر ماڈل کی موازنہ اور انتخاب
ہم کثیر مشین لرننگ ماڈلز کو تربیت دے سکتے ہیں اور ان کی کارکردگی کا موازنہ کر سکتے ہیں۔ کلاسیفیکیشن کے کاموں کے لیے کچھ مشہور ماڈلز میں شامل ہیں:

* لاگسٹک ریگریشن
* فیصلہ درخت
* رینڈم جنگل
* سپورٹ ویکٹر مشین (SVM)

یہاں ایک مثال ہے کہ ہم کس طرح Python کا استعمال کرتے ہوئے ان ماڈلز کو تربیت دے سکتے ہیں اور ان کی کارکردگی کا موازنہ کر سکتے ہیں:
```python
from sklearn.linear_model import LogisticRegression
from sklearn.tree import DecisionTreeClassifier
from sklearn.ensemble import RandomForestClassifier
from sklearn.svm import SVC
from sklearn.metrics import accuracy_score

# ماڈلز کو تربیت دیں
logreg = LogisticRegression()
logreg.fit(X_train, y_train)

dt = DecisionTreeClassifier()
dt.fit(X_train, y_train)

rf = RandomForestClassifier()
rf.fit(X_train, y_train)

svm = SVC()
svm.fit(X_train, y_train)

# ماڈلز کی کارکردگی کا موازنہ کریں
y_pred_logreg = logreg.predict(X_test)
y_pred_dt = dt.predict(X_test)
y_pred_rf = rf.predict(X_test)
y_pred_svm = svm.predict(X_test)

print("لاگسٹک ریگریشن کی درستی:", accuracy_score(y_test, y_pred_logreg))
print("فیصلہ درخت کی درستی:", accuracy_score(y_test, y_pred_dt))
print("رینڈم جنگل کی درستی:", accuracy_score(y_test, y_pred_rf))
print("SVM کی درستی:", accuracy_score(y_test, y_pred_svm))
```

### ہائپرپیرامیٹر ٹیوننگ کے ساتھ GridSearch/RandomSearch
ہائپرپیرامیٹر ٹیوننگ مشین لرننگ ماڈل کے لیے بہترین ہائپرپیرامیٹرز کو تلاش کرنے کا عمل ہے۔ ہم GridSearch یا RandomSearch کا استعمال کرتے ہوئے ہائپرپیرامیٹرز کو ٹیون کر سکتے ہیں۔ یہاں ایک مثال ہے کہ ہم کس طرح GridSearch کا استعمال کرتے ہوئے ہائپرپیرامیٹرز کو ٹیون کر سکتے ہیں:
```python
from sklearn.model_selection import GridSearchCV

# ہائپرپیرامیٹرز کی گرڈ کو تعریف کریں
param_grid = {
    'C': [0.1, 1, 10],
    'kernel': ['linear', 'rbf', 'poly']
}

# GridSearch کو کریں
grid_search = GridSearchCV(SVC(), param_grid, cv=5)
grid_search.fit(X_train, y_train)

# بہترین ہائپرپیرامیٹرز اور ان کی درستی کو پرنٹ کریں
print("بہترین ہائپرپیرامیٹرز:", grid_search.best_params_)
print("بہترین درستی:", grid_search.best_score_)
```

### ماڈل کی تعیناتی کی حکمت عملیاں
ہم نے ماڈل کو تربیت دی اور ٹیون کر لیا ہے، اب ہم اسے پروڈکشن ماحول میں تعینات کر سکتے ہیں۔ کچھ مشہور تعیناتی کی حکمت عملیاں میں شامل ہیں:

* ویب ایپلیکیشن کے طور پر ماڈل کو تعینات کرنا
* موبائل ایپلیکیشن کے طور پر ماڈل کو تعینات کرنا
* API کے طور پر ماڈل کو تعینات کرنا

```python
from fastapi import FastAPI

    data['max_nami'] = data['nami'].rolling(window=7).max()
    data['min_mati_nami'] = data['mati_nami'].rolling(window=7).min()

    # پیش گوئی کریں
    prediction = svm.predict(data)

```

### ریئل ٹائم مانیٹرنگ ڈیش بورڈ
ہم ریئل ٹائم مانیٹرنگ ڈیش بورڈ بنا سکتے ہیں تاکہ نظام کی مانیٹرنگ کریں اور بیماریوں کی تشخیص کریں۔ ڈیش بورڈ میں درج ذیل معلومات کو ظاہر کیا جا سکتا ہے:

* موجودہ درجہ حرارت، نمی، اور مٹی کی نمی کی پڑھائی
* پیش گوئی کی بیماری کی حیثیت
* تاریخی ڈیٹا اور رجحانات

یہاں ایک مثال ہے کہ ہم کس طرح Dash کا استعمال کرتے ہوئے ایک سادہ ڈیش بورڈ بنا سکتے ہیں:
```python
import dash
import dash_core_components as dcc
import dash_html_components as html
from dash.dependencies import Input, Output
import plotly.express as px

app = dash.Dash(__name__)

app.layout = html.Div([
    html.H1('ریئل ٹائم مانیٹرنگ ڈیش بورڈ'),
    dcc.Graph(id='drj_hararat-graph'),
    dcc.Graph(id='nami-graph'),
    dcc.Graph(id='mati_nami-graph'),
    html.Div(id='prediction-output')
])

@app.callback(
    Output('drj_hararat-graph', 'figure'),
    [Input('drj_hararat-graph', 'id')]
)
def update_drj_hararat_graph(input):
    # تازہ ترین درجہ حرارت ڈیٹا کو حاصل کریں
    data = pd.read_csv('drj_hararat_data.csv')

    # فیگچر کو بنائیں
    fig = px.line(data, x='Time', y='drj_hararat')

    return fig

@app.callback(
    Output('prediction-output', 'children'),
    [Input('drj_hararat-graph', 'id')]
)
def update_prediction_output(input):
    # تازہ ترین پیش گوئی ڈیٹا کو حاصل کریں
    prediction = svm.predict(pd.DataFrame([{'drj_hararat': 25, 'nami': 60, 'mati_nami': 40}]))

    return f"پیش گوئی کی بیماری کی حیثیت: {prediction}"
```

## عملی اطلاقات زراعت/پودوں کی سائنس میں
بیماری کی پیش گوئی کا نظام مختلف زرعی ماحول میں اطلاق کیا جا سکتا ہے، جیسے کہ:

* **گندم کی کاشت**: گندم کی فصلیں میں بیماریوں جیسے کہ پاؤڈری ملڈیو اور رسٹ کی پیش گوئی
* **چاول کی کاشت**: چاول کی فصلیں میں بیماریوں جیسے کہ بلاسٹ اور شیٹ بلائٹ کی پیش گوئی
* **ٹماٹر کی کاشت**: ٹماٹر کی فصلیں میں بیماریوں جیسے کہ ابتدائی بلیٹ اور دیر سے بلیٹ کی پیش گوئی

نظام کو دوسرے زرعی نظاموں کے ساتھ بھی یکجائی کیا جا سکتا ہے، جیسے کہ:

* **پریشن زراعت**: نظام کو پریشن زراعت کے ساتھ یکجائی کرکے فصلیں کی دیکھ بھال کو بہتر بنانا
* **زرعی ڈرونز**: ڈرونز کو استعمال کرتے ہوئے فصلیں کی مانیٹرنگ کرنا

## بہترین طریقے اور عام غلطیاں
کچھ بہترین طریقے جو بیماری کی پیش گوئی کے نظام کو بناتے وقت ذہن میں رکھنے چاہیئں:

* **ڈیٹا کی معیار**: یقینی بنائیں کہ ڈیٹا صحیح، مکمل، اور مستحکم ہے
* **ماڈل کا انتخاب**: مناسب مشین لرننگ ماڈل کا انتخاب کریں
* **ہائپرپیرامیٹرز کی ٹیوننگ**: ماڈل کی کارکردگی کو بہتر بنانے کے لیے ہائپرپیرامیٹرز کو ٹیون کریں
* **ماڈل کی تعیناتی**: ماڈل کو پروڈکشن ماحول میں تعینات کریں اور اس کی کارکردگی کو مانیٹر کریں

کچھ عام غلطیاں جو ٹھیک کرنی چاہیئں:

* **اوور فٹنگ**: ماڈل کو بہت زیادہ ڈیٹا پر تربیت دینا اور اس کو اوور فٹ کرنا
* **انڈر فٹنگ**: ماڈل کو بہت کم ڈیٹا پر تربیت دینا اور اس کو انڈر فٹ کرنا
* **ڈیٹا لیکج**: پیش گوئی کے وقت دستیاب نہ ہونے والے ڈیٹا کو استعمال کرنا

## عملی مثال یا مینی پروجیکٹ
آئیے ایک سادہ بیماری کی پیش گوئی کا نظام بنائیں جس میں ہم نے اس ماڈیول میں سیکھے ہوئے تصورات کو استعمال کیا ہے۔ ہم گندم کی فصلیں کے ڈیٹا سیٹ کو استعمال کریں گے اور بیماری کی پیش گوئی کریں گے۔

یہاں کوڈ ہے:
```python
import pandas as pd
from sklearn.model_selection import train_test_split
from sklearn.linear_model import LogisticRegression
from sklearn.metrics import accuracy_score

# ڈیٹا سیٹ کو لوڈ کریں
df = pd.read_csv('gandum_data.csv')

# ڈیٹا کی پری پروسیسنگ
df = df.dropna()  # خالی مقامات کو ہٹا دیں
df = df.drop_duplicates()  # دہرے مقامات کو ہٹا دیں

# ڈیٹا کو تربیت اور ٹیسٹ سیٹ میں تقسیم کریں
X_train, X_test, y_train, y_test = train_test_split(df.drop('bimari_ki_jaga', axis=1), df['bimari_ki_jaga'], test_size=0.2, random_state=42)

# لاگسٹک ریگریشن ماڈل کو تربیت دیں
logreg = LogisticRegression()
logreg.fit(X_train, y_train)

# ماڈل کی درستی کو چیک کریں
y_pred = logreg.predict(X_test)
print("درستی:", accuracy_score(y_test, y_pred))
```

## خلاصہ ٹیبل یا چیک لسٹ
یہاں ایک خلاصہ ٹیبل ہے جو اس ماڈیول میں سیکھے ہوئے تصورات کو ظاہر کرتا ہے:

| تصور | وضاحت |
| --- | --- |
| ڈیٹا کی جمع | سنسورز اور ماحولیاتی عوامل سے ڈیٹا کی جمع |
| ڈیٹا کی پری پروسیسنگ | جمع کیے گئے ڈیٹا کی صفائی اور پری پروسیسنگ |
| فیچر انجینئرنگ | پری پروسس کیے گئے ڈیٹا سے متعلقہ خصوصیات کی اختراع |
| ماڈل کی تربیت | انجینئرڈ خصوصیات کا استعمال کرتے ہوئے مشین لرننگ ماڈل کی تربیت |
| ماڈل کی تعیناتی | تربیت یافتہ ماڈل کی تعیناتی پروڈکشن ماحول میں |
| ریئل ٹائم مانیٹرنگ | نظام کی ریئل ٹائم مانیٹرنگ بیماریوں کی تشخیص کے لیے |

یہاں ایک چیک لسٹ ہے جو بیماری کی پیش گوئی کے نظام کو بنانے کے لیے ضروری اقدامات کو ظاہر کرتا ہے:

1. سنسورز اور ماحولیاتی عوامل سے ڈیٹا کی جمع
2. ڈیٹا کی پری پروسیسنگ
3. پری پروسس کیے گئے ڈیٹا سے متعلقہ خصوصیات کی اختراع
4. انجینئرڈ خصوصیات کا استعمال کرتے ہوئے مشین لرننگ ماڈل کی تربیت
5. تربیت یافتہ ماڈل کی تعیناتی پروڈکشن ماحول میں
6. نظام کی ریئل ٹائم مانیٹرنگ بیماریوں کی تشخیص کے لیے

## اگلا قدم اور مزید پڑھائی
کچھ اگلا قدم جو ہم لے سکتے ہیں:

* **دوسرے زرعی نظاموں کے ساتھ یکجائی**: بیماری کی پیش گوئی کے نظام کو دوسرے زرعی نظاموں کے ساتھ یکجائی کرنا
* **دوسرے مشین لرننگ ماڈلز کا استعمال**: دوسرے مشین لرننگ ماڈلز کا استعمال کرکے بیماری کی پیش گوئی کے نظام کی درستی کو بہتر بنانا
* **مزید ڈیٹا کی جمع**: مزید ڈیٹا کی جمع کرکے بیماری کی پیش گوئی کے نظام کی درستی کو بہتر بنانا

کچھ مزید پڑھائی کے لیے مواد:

* **"مشن لرننگ برائے زراعت"**: ایک کتاب جو زراعت میں مشین لرننگ کے اطلاق کو ظاہر کرتی ہے
* **"بیماری کی پیش گوئی فصلیں میں"**: ایک تحقیقی مقالہ جو فصلیں میں بیماری کی پیش گوئی کے لیے مشین لرننگ کے استعمال کو ظاہر کرتا ہے
* **"پریشن زراعت"**: ایک کتاب جو پریشن زراعت کے تصور اور اس کے زراعت میں اطلاق کو ظاہر کرتی ہے
